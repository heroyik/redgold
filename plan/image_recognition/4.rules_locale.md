# Rules for Localization (KO & JA)

이 규칙은 `lesson{N}.json` 파일 내에 한국어(KO) 및 일본어(JA) 번역 데이터를 포함하여, 코드 내 하드코딩 없이 다국어를 지원하기 위한 표준 가이드 및 구현 로직을 정의합니다.

## 1. 개요 및 목적
기존에는 `src/utils/lessonTranslations.ts` 파일에 모든 레슨의 번역이 중앙 집중식으로 하드코딩되어 있었습니다. 이는 레슨이 늘어날수록 파일 크기가 비대해지고 관리 효율이 떨어지는 문제를 야기합니다.
이를 해결하기 위해 **데이터(JSON)와 번역을 하나로 통합**하고, 코드는 이를 동적으로 처리하는 구조로 변경합니다.

## 2. JSON 구조 표준
모든 `lesson{N}.json` 파일의 루트 레벨에 `translations` 객체를 추가합니다.

```json
{
  "lessonId": 1,
  "title": "简单的爱情",
  "vocabulary": [...],
  "translations": {
    "vocabulary": {
      "法律": { "ko": "법률", "ja": "法律" }
    },
    "properNouns": {
      "孙月": { "ko": "쑨웨", "ja": "孫月" }
    },
    "grammar": {
      "不仅……也/还/而且……": {
        "formal_translation": { "ko": "...", "ja": "..." },
        "colloquial_translation": { "ko": "...", "ja": "..." }
      }
    },
    "keySentences": {
      "虽然我们认识的时间不长...": {
        "translation": { "ko": "...", "ja": "..." },
        "colloquial_translation": { "ko": "...", "ja": "..." },
        "context": { "ko": "...", "ja": "..." }
      }
    },
    "texts": {
      "1": {
        "lines": {
          "0": { "translation": { "ko": "...", "ja": "..." } }
        }
      }
    }
  }
}
```

## 3. 동적 로컬라이징 로직 (Refactoring)

`translateLessonData` 함수는 더 이상 외부의 static 객체를 참조하지 않고, 전달받은 `lessonData` 내부의 `translations` 필드를 사용하도록 수정됩니다.

### 변경된 로직 흐름
1. `App.ts`에서 `lesson{N}.json`을 로드합니다.
2. 로드된 데이터 객체(`_data`)를 `translateLessonData`에 전달합니다.
3. 함수는 `lessonData.translations`가 존재하는지 확인하고, 설정된 언어(`ko` 또는 `ja`)에 맞춰 원본 필드(`meaning`, `translation` 등)를 덮어씁니다.

### 코드 예시 (Concept)
```typescript
export function translateLessonData(lessonData: any, language: AppLanguage) {
  const translations = lessonData.translations;
  if (!translations || language === 'en') return lessonData;

  // translations 객체를 순회하며 vocabulary, grammar, texts 등을 매핑
  // 예: item.meaning = translations.vocabulary[item.word][language] || item.meaning
}
```

## 4. 번역 품질 및 콘텐츠 가이드라인

### 한국어 (KO)
- **자연스러움**: 한자어나 중국어 직역보다는 한국어 화자가 일상에서 사용하는 자연스러운 문맥을 우선합니다.
- **존칭**: 교과서 표현(Formal)은 '해요체'를 기본으로 하며, 구어체(Colloquial)는 상황에 따라 친근한 말투를 사용합니다.
- **의역**: 중국어 관용구는 한국어의 대응되는 표현으로 의역합니다.

### 일본어 (JA)
- **한자 혼용**: 학습 수준을 고려하여 적절한 한자와 가나 비중을 유지합니다.
- **어투**: Formal은 'です/ます', Colloquial은 'だ/である' 또는 실제 회화체를 사용합니다.

## 5. 마이그레이션 및 워크플로우

1.  **데이터 추출**: `src/utils/lessonTranslations.ts`에서 해당 레슨의 번역 데이터를 복사합니다.
2.  **JSON 업데이트**: `data/lesson{N}.json` 파일 하단에 `translations` 필드를 생성하고 데이터를 붙여넣습니다.
3.  **코드 수정**: `translateLessonData` 함수를 위에서 정의한 동적 로직으로 업데이트합니다.
4.  **AI 자동화**: 새로운 레슨 이미지를 인식할 때, AI는 자동으로 `translations` 블록까지 포함된 최종 JSON을 생성해야 합니다.
5.  **검증**: `translations`의 Key(원본 텍스트)가 JSON의 실제 데이터 필드와 100% 일치하는지 확인합니다.
