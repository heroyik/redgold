# 🇨🇳 Kewen(课文) JSON 생성 규칙 (AI Sidekick)

## 🎯 목적
중국어 학습 자료의 **'课文(본문/대화)'** 이미지를 분석하여, `data/lesson{번호}.json`의 `texts` 배열을 프로젝트 표준에 맞춰 최적화된 데이터로 생성하거나 업데이트한다.

## 🛠️ 핵심 요구사항 및 로직
1.  **데이터 구조 (JSON Schema)**:
    *   `id`: 해당 레슨 내 텍스트 순서 (1부터 시작).
    *   `title`: 중문 제목 + (영문/국문 번역). 예: "孙月和王静聊王静의男朋友 (Sun Yue and Wang Jing talk about Wang Jing's boyfriend)"
    *   `audio`: `audio/{lessonId}-{textId}.mp3` 형식. (예: `audio/01-1.mp3`)
    *   `content`: 화자별 대화 목록.
        *   `speaker`: 화자 이름. 독백인 경우 "独白".
        *   `text`: 본문 한자. **핵심 문법 패턴은 `<b>` 태그로 강조.**
        *   `pinyin`: 성조가 포함된 정확한 병음.
        *   `translation`: 자연스러운 한국어/영어 번역.
    *   `proper_nouns`: 이름, 지명 등 고유 명사 목록 (`word`, `pinyin`, `meaning`).
    *   `vocabulary`: 해당 텍스트 하단에 제시된 핵심 단어 목록 (`word`, `pinyin`, `meaning`).

2.  **오디오 동기화 로직 최적화**:
    *   프로젝트의 `TextSection.ts`는 한자 수와 병음 길이를 기준으로 오디오 하이라이트 타이밍을 자동 계산한다.
    *   따라서 `text`와 `pinyin` 필드는 실제 음성 속도와 일치하도록 누락 없이 정확하게 작성해야 한다.

3.  **하이라이트 스타일링 (`<b>`)**:
    *   이미지에서 해당 레슨의 핵심 문법이 사용된 부분을 찾아 반드시 `<b>` 태그를 적용한다.
    *   UI에서 이 태그는 자동으로 **Imperial Red (#8B0000)**로 스타일링된다.

## ⚙️ 상세 처리 가이드
1.  **이미지 OCR 및 분석**: 텍스트 번호, 제목, 화자, 대화 내용, 하단 단어장을 정확히 분리한다.
2.  **병음 생성 표준**: 
    *   단어별 띄어쓰기를 준수한다 (예: `nǐ hǎo` vs `nǐhǎo`).
    *   문장 부호 뒤에는 공백을 둔다.
3.  **번역 퀄리티**: 교과서적인 딱딱한 번역보다는 한국인/영미권 사용자가 읽었을 때 상황이 바로 이해되는 자연스러운 문체로 작성한다.
4.  **JSON 업데이트 정책**:
    *   기존 파일을 읽어 해당 `id`를 가진 항목이 있으면 교체(Replace)하고, 없으면 추가(Append)한다.
    *   `lessonId`와 `title`이 유효한지 상단에서 먼저 확인한다.
