# 🇨🇳 Notes(注释) & Vocabulary JSON 생성 규칙 (AI Sidekick)

## 🎯 목적
학습 자료의 **'注释(문법/노트)'** 및 **'生词(새 단어)'** 이미지를 분석하여, `grammar` 및 `vocabulary` 배열을 풍성하고 실전적인 데이터로 생성한다.

## 🛠️ 핵심 요구사항 및 로직
1.  **Grammar(문법) 데이터 구조**:
    *   `point`: 문법 핵심 표현 (예: "不仅……也/还……")
    *   `explanation`: 용법에 대한 명확한 설명.
    *   **Formal Side (표준 - 교재 예문 전부 포함)**:
        *   `formal_examples`: 이미지에 등장하는 **모든 예문**을 객체 배열로 수집.
            *   `chinese`: 교재 예문 원문. 패턴은 `<b>` 태그로 강조.
            *   `pinyin`: 예문 병음.
            *   `translation`: 예문 번역.
    *   **Colloquial Side (실전 구어체 - Enrichment)**:
        *   `colloquial_pattern`: 원어민이 실제로 더 자주 쓰는 대체 패턴 (예: `不仅` -> `不光`).
        *   `colloquial_version`: **AI가 독자적으로 생성한 실전 문장.** `贼`, `给力`, `领证` 등 트렌디하고 생생한 어휘 사용 권장. 패턴은 `<b>` 태그로 강조.
        *   `colloquial_pinyin`: 구어체 문장 병음.
        *   `colloquial_translation`: 구어체 문장 번역.
    *   `nuance`: 표준 버전과 구어체 버전의 미묘한 차이 설명 (예: "친구 사이에서 훨씬 더 감정을 실어 표현할 때 사용").

2.  **Vocabulary(단어) 데이터 구조**:
    *   `word`: 단어 한자.
    *   `pinyin`: 정확한 성조 병음.
    *   `meaning`: 의미. 품사 정보(n., v., adj.)가 있으면 접두어로 포함 (예: "n. law").

3.  **UI 렌더링 호환성**:
    *   `GrammarCard.ts`는 앞면(Formal)과 뒷면(Colloquial)을 뒤집으며 보여주는 구조이다.
    *   뒷면의 `<b>` 태그는 CSS에 의해 자동으로 **Gold (#DAA520)** 색상으로 표시되므로, 강조하고 싶은 구어체 패턴에 반드시 적용한다.

## ⚙️ 상세 처리 가이드
1.  **이미지 분석**: 문법 포인트의 번호와 제목, 설명을 정확히 매칭하며, **교재에 적힌 모든 예문을 빠짐없이 `formal_examples` 배열에 담는다.**
2.  **창의적 구어체 생성**: 단순히 교과서 문장을 바꾸는 수준을 넘어, 해당 상황에서 원어민이 느낄 법한 감정이나 최신 트렌드를 반영한 문장을 생성하라. (구어체는 가장 대표적인 예문 하나를 기반으로 생성하거나 포인트 자체를 활용한다.)
3.  **데이터 정규화**:
    *   병음 성조 기호(ā, á, ǎ, à)를 정확히 사용한다.
    *   불필요한 공백이나 특수문자를 제거하여 JSON 순수성을 유지한다.
4.  **병합 로직**:
    *   새로운 단어는 `vocabulary` 배열 끝에 추가하되 중복을 체크한다.
    *   문법은 `point`를 기준으로 기존 데이터를 업데이트하거나 새 항목을 생성한다.

5.  **대화문(A/B) 처리 규칙**:
    *   예문에 대화 패턴(`A: ..., B: ...`)이 등장할 경우, 가독성을 위해 **`<br/>` 태그**를 사용하여 줄바꿈을 처리한다.
    *   `chinese`, `pinyin`, `translation` 모든 필드에서 대화 주체가 바뀔 때마다(B: 시작 전) `<br/>`를 삽입한다.
