# 중국어 단어장 앱 - 상세 작업 항목 및 To-Do 리스트 (Detailed Tasks & To-Do List)

이 문서는 프로젝트의 요구사항 정의서와 구현 계획서를 바탕으로, 개발 과정에서 실질적으로 수행해야 할 세부 작업 항목들을 정리합니다.

## 📋 프로젝트 목표 재확인
* **최종 목표:** HSK 4급 구어체 학습에 특화된, 모바일 우선(Mobile-First)의 고품질 중국어 단어장 앱 개발.
* **개발 방식:** 린 개발(Lean Development) 기반의 MVP 출시 및 반복 개선.

---

## 🎯 Phase 0: 기획 및 설계 (Design & Blueprinting) - 현재 초점 영역
**목표:** 모든 핵심 구조와 디자인 가이드를 확정하여 개발 착수 준비 완료.

### 🎨 UX/UI 디자인 및 스타일 가이드
* [ ] **와이어프레임 제작:** 핵심 화면 (로그인, 학습 메인, 콘텐츠 보기, 복습/오답 노트, 프로필)에 대한 저화질 와이어프레임 제작.
* [ ] **스타일 가이드 정의:**
    * **톤 앤 매너:** 미니멀리즘/플랫 디자인 기반의 세련되고 차분한 색상 팔레트 및 폰트 계층 구조 확정.
    * **가독성 원칙:** 한자(가장 중요) -> 병음 -> 한국어 번역의 시각적 위계(Hierarchy)를 명확히 하는 디자인 요소 정의.
    * **모바일 최적화:** 최신 스마트폰 비율(20:9 이상)을 기준으로 모든 레이아웃 테스트 및 조정.
* [ ] **사용자 플로우 정의:** 사용자가 앱을 켜는 순간부터 목표를 달성하는(예: 단어 10개 학습 완료) 전체 여정(User Flow)을 다이어그램으로 완성.

### 🗄️ 콘텐츠 및 데이터 모델링
* [ ] **핵심 데이터 모델 설계 (Schema):**
    * **단어(Word):** 단어 자체, HSK 레벨, 병음, 품사, 한국어 의미.
    * **구문(Phrase):** 구어체/문어체 구분, 전체 문장, 핵심 단어 연결, 사용 예시.
    * **문맥(Context):** 이미지 캡처 파일 경로, 원문(중국어), 본문 핵심 단어 목록, 출처(교재명).
    * **문법/패턴(Grammar):** 문법 규칙, 예문, 관련 단어 링크.
    * **구어체(Colloquial):** 원어민 표현, 사용 상황(Context), 의미.
* [ ] **관리자 시스템 API 정의:** 관리자가 위 데이터 모델을 쉽게 CRUD(Create, Read, Update, Delete)할 수 있도록 API 엔드포인트 명세서 작성.

## 🛠️ Phase 1: MVP (Minimum Viable Product) 구현
**목표:** 핵심 학습 루프를 완성하고, 구어체 학습의 가치를 빠르게 검증.

### 💻 백엔드 구축
* [ ] **API 구축:** Phase 0에서 설계한 데이터 모델을 기반으로 백엔드 API를 실제로 구축 (Authentication, Content Retrieval API 등).
* [ ] **초기 콘텐츠 로딩:** HSK 4급 단어 100개와 구어체 표현 50개를 임시로 DB에 로딩하여 테스트 가능 상태로 만듦.

### 📱 프론트엔드 개발
* [ ] **기본 학습 화면 구현:** 단어 제시 -> 의미 확인 -> 발음 듣기 -> 퀴즈 풀기 (최소 기능).
* [ ] **구어체 노출 섹션 구현:** 본문과 관련된 구어체 표현을 별도의 탭이나 모듈로 노출하는 인터페이스 구현.
* [ ] **진행도 표시:** 사용자가 현재까지 학습한 단어/구문 개수를 시각적으로 보여주는 기능 추가.
