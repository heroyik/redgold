# Localization Rules (Locale)

## Overview
To support multi-language (Korean, Japanese) without hardcoding strings in the source code, all translations are managed within the lesson data JSON files (`data/lesson*.json`).

## Data Structure
Each lesson JSON file contains a `translations` object at the top level with the following structure:

```json
{
  "translations": {
    "vocabulary": {
      "WORD": { "ko": "...", "ja": "..." }
    },
    "properNouns": {
      "NAME": { "ko": "...", "ja": "..." }
    },
    "grammar": {
      "POINT": {
        "formal_translation": { "ko": "...", "ja": "..." },
        "colloquial_translation": { "ko": "...", "ja": "..." }
      }
    },
    "keySentences": {
      "CHINESE_SENTENCE": {
        "translation": { "ko": "...", "ja": "..." },
        "colloquial_translation": { "ko": "...", "ja": "..." },
        "context": { "ko": "...", "ja": "..." }
      }
    },
    "texts": {
      "TEXT_ID": {
        "lines": {
          "LINE_INDEX": {
            "translation": { "ko": "...", "ja": "..." }
          }
        }
      }
    }
  }
}
```

## Logic
The utility function `translateLessonData` in `src/utils/lessonTranslations.ts` is responsible for mapping these translations onto the core lesson data.

1. **Mapping Vocabulary**: Uses `item.word` as the key.
2. **Mapping Grammar**: Uses `item.point` as the key.
3. **Mapping Key Sentences**: Uses `item.sentence` as the key.
4. **Mapping Texts**: Uses `text.id` and the line's `index` to find the translation.

## Rules
1. **No Hardcoding**: Do not add Korean or Japanese strings directly in the TypeScript files.
2. **Key Consistency**: Ensure that the keys in `translations` (like Chinese sentences or grammar points) exactly match the fields in the main lesson data.
3. **Unified Entries**: Each translation entry must contain both `ko` and `ja` fields to ensure consistency across languages.
